# AI助手未取货买家信息获取优化计划

## 问题分析

通过代码分析，发现AI只显示了Ymc的未取货订单，而没有显示其他买家（如Ls）的未取货订单的原因：

1. **问题1：商品匹配逻辑可能存在问题**
   - `matchesGoodsIdentity` 函数可能无法正确匹配所有购买该商品的订单
   - 导致只匹配到了Ymc的订单，而没有匹配到Ls的订单

2. **问题2：订单获取逻辑需要优化**
   - 目前的实现可能只获取了部分订单数据
   - 需要确保获取所有未取货的订单

## 解决方案

### 1. 优化 `merchant_pending_pickup_goods` 意图处理
- 直接使用 `getMerchantOrderGoods` 云函数的 `customer` 类型获取所有买家的未取货订单
- 按买家分组显示，包含所有买家的未取货订单信息
- 确保回复格式符合用户要求

### 2. 优化 `merchant_goods_buyers` 意图处理
- 确保使用正确的商品ID进行匹配
- 从商品详情中提取所有购买该商品且未取货的买家信息
- 确保回复格式符合用户要求

## 实施步骤

1. **修改 `cloudfunctions/aiAssistant/index.js` 文件**
   - 修改 `merchant_pending_pickup_goods` 意图的处理逻辑，使用 `getMerchantOrderGoods` 云函数的 `customer` 类型
   - 确保获取所有未取货的买家信息

2. **测试验证**
   - 测试"未取货的订单有哪些"查询
   - 测试"水果玉米还有谁没取货"查询
   - 验证回复是否包含所有买家的未取货订单信息

## 预期结果

1. 当询问"未取货的订单有哪些"时，AI应回复所有待取货的订单，包括所有买家的信息，如：
   ```
   以下买家有货品还没有取，详情如下
   Ls  23455553321 取货码:201208
   水果玉米   ×3
   Ymc  124435566 取货码:633116
   水果玉米   ×5
   ```

2. 当询问"水果玉米还有谁没取货"时，AI应回复所有购买水果玉米且未取货的买家信息，如：
   ```
   Ls 23455553321 还有3件水果玉米未取；Ymc 124435566 还有5件水果玉米未取
   ```
