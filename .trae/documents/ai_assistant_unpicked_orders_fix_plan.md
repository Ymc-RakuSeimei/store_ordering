# AI助手未取货订单查询优化计划

## 问题分析

通过代码分析，发现AI助手在处理未取货订单查询时存在以下问题：

1. **问题1：意图识别错误**

   * 当用户问"未取货的订单有哪些"时，`detectIntent` 函数会匹配到 `merchant_unpicked` 意图

   * 但 `merchant_unpicked` 意图只处理"超过3天未取"的订单

   * 而 `merchant_pending_pickup_goods` 意图才是处理所有未取货订单的

2. **问题2：回复格式不符合要求**

   * `merchant_pending_pickup_goods` 意图的回复格式不符合用户要求

   * 用户希望的格式是按买家人名分别回复，显示所有未取货的商品，同时也要包含取货码

## 解决方案

### 1. 优化 `detectIntent` 函数

* 调整正则表达式匹配顺序和规则

* 确保"未取货的订单有哪些"匹配到 `merchant_pending_pickup_goods` 意图

* 确保"目前滞留的订单有哪些"匹配到 `merchant_unpicked` 意图

### 2. 优化 `merchant_pending_pickup_goods` 意图的回复格式

* 修改回复格式，使其符合用户要求：按买家人名分别回复，显示所有未取货的商品

* 去掉下单时间和取货码，只显示买家信息和商品信息

## 实施步骤

1. **修改** **`cloudfunctions/aiAssistant/logic.js`** **文件**

   * 修改 `detectIntent` 函数，调整意图识别规则

   * 修改 `merchant_pending_pickup_goods` 意图的回复格式

2. **测试验证**

   * 测试"未取货的订单有哪些"查询

   * 测试"目前滞留的订单有哪些"查询

   * 验证回复格式是否符合要求

## 预期结果

1. 当询问"未取货的订单有哪些"时，AI应回复：

   ```
   以下买家有货品还没有取，详情如下
   Ls  23455553321 取货码:201208
   水果玉米   ×3
   Ymc  124435566  取货码:633116
   水果玉米   ×5
   ```

2. 当询问"目前滞留的订单有哪些"时，AI应回复超过3天未取的订单

## 风险评估

* **低风险**：修改仅限于优化意图识别和回复格式，不影响其他功能

* **兼容性**：保持与现有API和数据结构的兼容性

* **性能**：新增逻辑复杂度低，不会影响响应速度
