# 首页UI布局优化计划

## 1. 代码库研究结论

通过对代码库的分析，我发现：

### 买家端首页 (customer/index/index)

* 布局结构：顶部欢迎区域 + 卡片容器 + 长卡片（消息中心） + 底部文字

* 卡片容器：包含4个功能卡片和1个AI智能助手入口（位于中央）

* 长卡片：消息中心，包含图标、标题和未读消息徽章

* AI入口：位于卡片容器中央，圆形设计，蓝色背景，白色边框

### 商家端首页 (merchant/index/index)

* 布局结构：顶部欢迎区域 + 卡片容器 + 长卡片（通知） + 底部文字 + 底部导航栏

* 卡片容器：包含多个功能卡片和1个AI智能助手入口（位于中央）

* 长卡片：通知中心，包含图标、标题、副标题和通知徽章

* AI入口：位于卡片容器中央，圆形设计，蓝色背景，白色边框

## 2. 优化目标

根据用户要求，优化首页UI布局，具体目标：

* 将消息中心所在位置的长矩形分为左右两块矩形

* 左边仍然是"消息中心"（买家端）或"通知"（商家端）

* 右边的矩形是"AI智能助手"

* 两块矩形样式一致

* 移除原来位于卡片容器中央的AI智能助手入口

## 3. 优化方案

### 3.1 买家端首页布局优化

**文件**：`miniprogram/pages/customer/index/index.wxml`
**文件**：`miniprogram/pages/customer/index/index.wxss`

**优化内容**：

1. **移除卡片容器中的AI入口**：删除位于卡片容器中央的AI智能助手入口
2. **修改长卡片结构**：将原来的单一长卡片改为包含左右两个子卡片的容器
3. **左边卡片**：保持消息中心的功能和样式
4. **右边卡片**：添加AI智能助手入口，样式与左边卡片一致
5. **调整样式**：确保左右卡片样式一致，包括背景、边框、阴影等。同时左右两个矩形的宽度与其上方对应的大矩形卡片哦

### 3.2 商家端首页布局优化

**文件**：`miniprogram/pages/merchant/index/index.wxss`

## 6. 风险评估

### 6.1 潜在风险

1. **样式冲突**：可能与现有的样式规则产生冲突
2. **布局问题**：左右卡片的布局可能在不同设备上显示不一致
3. **功能问题**：移除原有的AI入口可能影响功能

### 6.2 风险应对措施

1. **样式冲突**：使用更具体的选择器，确保样式规则优先级正确
2. **布局问题**：使用flexbox布局，确保在不同设备上的一致性
3. **功能问题**：确保新的AI入口具有与原入口相同的功能

## 8. 总结

通过本次优化，将使买家端和商家端首页的布局更加合理，将消息中心和AI智能助手放在同一行，样式一致，提升用户体验。优化后的布局将更加符合现代UI设计趋势，为用户提供更加美观、直观的界面。
