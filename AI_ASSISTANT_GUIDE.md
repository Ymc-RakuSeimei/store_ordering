# AI智能店员接入与维护说明

## 当前实现

- 首页中央悬浮 `AI` 图标，买家端和商家端位置统一，点击进入统一聊天页。
- 聊天页先调用云函数 `aiAssistant` 查库、识别意图、执行受控操作，再调用微信云开发大模型生成自然语言回复。
- 商家端支持查询营业额、接龙统计、低库存、补货建议、未取货订单、售后申请，并支持部分商品操作。
- 买家端支持查询商品价格/规格、库存、接龙活动、个人记录、订单状态、取货码和促销商品。
- 所有 AI 关键操作会写入 `ai_operation_logs` 集合，便于追溯。

## 需要你补充的内容

### 1. 开通微信云开发 AI 能力

请在微信云开发控制台确认当前环境已经开通 AI 能力，并可使用官方示例里的：

- `wx.cloud.extend.AI.createModel('deepseek')`
- 模型名 `deepseek-v3.2`

如果你的小程序基础库或云开发环境暂时没有这个能力，页面会自动退回为“数据库直出答复”，不会影响主要查询。

### 2. 如果你想改模型

修改文件：

- `miniprogram/utils/ai-assistant.js`

只需要改这两个常量：

- `AI_PROVIDER`
- `AI_MODEL`

### 3. 如果你要扩展更多 AI 指令

核心位置：

- `cloudfunctions/aiAssistant/logic.js`

主要扩展点：

- `detectIntent`
- `buildCustomerResult`
- `buildMerchantResult`

### 4. 商家操作能力说明

当前已接入：

- 修改价格
- 上架/下架商品
- 批量设置库存预警阈值

如果你后续要增加更高风险操作，例如删除商品、批量改库存、自动退款，建议改成“先识别意图，再弹确认框，再调用专用云函数”。

## 建议的验收问题

- 买家端：
  - `瑞幸咖啡多少钱`
  - `现在有哪些接龙活动`
  - `我的订单到货了吗`
  - `我的取货码是多少`
  - `今天有什么特价商品`

- 商家端：
  - `本周营业额是多少`
  - `哪些商品低库存`
  - `给我补货建议`
  - `未取货订单有哪些`
  - `有哪些新的售后申请`
  - `把瑞幸咖啡价格改成16元`
  - `把全部商品库存预警阈值设为8`

## 安全建议

- 不要把云数据库权限直接开放给前端查询敏感订单数据，统一通过云函数做权限校验。
- 如需对外审计，直接查看集合 `ai_operation_logs`。
- 如需更严格的数据保护，建议后续把手机号、取货码等敏感字段在进入模型前做更细粒度脱敏。

















# Ymc 4/9 AI优化

## 这次AI助手优化的总结

---

### 一、架构理解

当前AI助手采用 **「规则匹配 + 数据库查询 + 大模型生成」** 的混合架构：

```
用户输入 → 规则匹配意图 → 数据库查询 → 构建结构化数据 → 大模型生成回复
                ↓                                    ↓
          低成本快速响应                      自然语言优化
```

---

### 二、P0-1 优化：Prompt精简

**核心原理**：大模型对话中，系统提示词的token消耗是主要成本之一。精简Prompt能显著降低消耗而不影响效果。

**技术要点**：

| 要点         | 说明                                           |
| ------------ | ---------------------------------------------- |
| **角色归一** | 把多重描述合并为一句核心角色定义               |
| **指令收敛** | 把12+条分散指令收敛为2-3条核心要求             |
| **数据内嵌** | 把数据直接内嵌到prompt中，而不是让大模型"回忆" |
| **结构简化** | 去掉冗余的格式符号，用简洁的自然语言           |

**代码核心**：

```javascript
// 优化前：冗长分散
const systemContent = [
  `你是微信小程序"MC_store"的AI智能店员，当前服务对象是${role}。`,
  `你的职责是帮助商家管理店铺，查询经营数据，执行商品管理操作。`,
  `请基于提供的信息，用自然、友好、口语化的方式回答用户问题。`,
  // ... 10+ 条类似指令
];

// 优化后：归一简洁
const roleDesc = role === 'merchant'
  ? '你是MC_store商家助手，可查营业额/利润、库存、订单、售后；可执行改价、上下架、设置预警操作。'
  : '你是MC_store买家助手，只能查商品、订单、取货码、接龙、优惠，只能看自己的数据。';
```

---

### 三、P0-2 优化：幻觉检测

**核心原理**：大模型可能"自信地编造数据"，需要在输出环节进行 **一致性校验**，发现异常时修正。

**技术要点**：

| 要点             | 说明                                              |
| ---------------- | ------------------------------------------------- |
| **数据指纹提取** | 从原始数据和LLM回复中分别提取金额、数量等关键数据 |
| **一致性比对**   | 比对回复中的数据是否超出原始数据的合理范围        |
| **模式匹配检测** | 检测"原始数据空但回答有统计"等异常模式            |
| **自动修正**     | 对检测到的幻觉内容进行原位替换                    |

**核心算法流程**：

```javascript
// 1. 提取原始数据的特征
function extractDataFacts(data) {
  // 提取最大金额、数量、数据条数等
}

// 2. 提取回答中的数据特征  
function extractAnswerFacts(answer) {
  // 从回答中解析金额、数量
}

// 3. 检测一致性
function detectDataFabrication(originalData, llmAnswer) {
  if (原始数据为空 && 回答中有统计数据) → 警告
  if (回答中金额 > 原始最大金额 * 2) → 警告
}

// 4. 自动修正
function correctFabricatedAnswer(originalData, llmAnswer, warnings) {
  // 将编造的数字替换为"暂无相关数据"等安全表述
}
```

**验证点设计**：

| 场景         | 检测规则                                                    |
| ------------ | ----------------------------------------------------------- |
| 空数据被编造 | `原始数据长度 < 10` 且 `回答长度 > 30` 且包含"共/总计/一共" |
| 金额夸大     | `回答金额 > 原始最大金额 * 2`                               |
| 越权读取     | 买家视角出现"其他用户"/"别的订单"                           |

---

### 四、架构改进点

**前后端协同验证**：

```
云函数端（后）                          前端
    │                                    │
    ├─ 查询数据库                        │
    ├─ 构建 answerDraft ──────────────→ ├─ 获得 validationData
    │                                    │
    │                              ├─ 调用大模型
    │                              ├─ 获取回复
    │                              │
    │                              └─ detectDataFabrication()
    │                                        │
    │                                        ├─ 检测通过 → 直接展示
    │                                        └─ 检测失败 → correctFabricatedAnswer()
```
