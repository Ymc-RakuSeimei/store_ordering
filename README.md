# 朝夕优购-**面向社区及农村超市的“预售+自提”订货小程序**

## 项目介绍

这是一个基于微信小程序和云开发的商店小型超市订货取货系统，支持商家和顾客两种角色，提供商品管理、订单管理、预订单管理、消息通知、AI助手等功能。

## 功能特点

### 顾客端功能

- 首页浏览
- 商品列表与详情查看
- 新商品查看
- 购物车与结账
- 订单管理（待取货、已完成），取货码生成，
- 消息通知（全部、新商品、取货通知）
- 反馈提交
- 个人中心

### 商家端功能

- 商品管理（添加、编辑、删除）
- 订单管理与详情查看
- 预订单创建与管理
- 数据中心统计
- 通知管理（反馈回复）
- 取货码验证
- 个人中心

### 其他功能

- AI助手
- 消息标记已读
- 取货提醒

## 技术栈

- **前端**：微信小程序原生开发

- **后端**：微信云开发（云函数）

- **数据库**：微信云数据库

## 安装部署

### 前置条件

- 微信开发者工具
- 微信小程序账号
- 微信云开发环境

### 部署步骤

1. **克隆项目**

   ```bash
   git clone <repository-url>
   cd store_ordering
   ```

2. **导入项目到微信开发者工具**

   - 打开微信开发者工具
   - 点击「导入项目」
   - 选择项目目录
   - 填写小程序 AppID（在 project.config.json 中查看）

3. **配置云开发环境**

   - 在微信开发者工具中，点击「云开发」按钮
   - 创建或选择已有的云开发环境
   - 在 `miniprogram/envList.js` 中配置云环境 ID

4. **部署云函数**

   - 在微信开发者工具中，右键点击 `cloudfunctions` 目录

   - 选择「上传并部署所有云函数」

   - 或使用脚本上传：

     ```bash
     sh uploadCloudFunction.sh
     ```

5. **构建 npm**

   - 在微信开发者工具中，点击「工具」->「构建 npm」

6. **预览与发布**

   - 点击「预览」按钮生成预览码
   - 测试无误后，点击「上传」按钮发布小程序

## 使用说明

### 顾客端

1. **注册/登录**：首次使用时，系统会自动创建用户信息(点击首页/“我的”右上角头像位置)
2. **浏览商品**：在「商品」页面查看商品列表
3. **下单**：选择商品后进入结账页面，确认订单信息
4. **查看订单**：在「我的」页面点击「我的订单」查看订单状态
5. **取货**：凭取货码到店取货
6. **提交反馈**：在「我的」页面点击「反馈」提交意见

### 商家端

1. **切换角色**：在「我的」页面点击「切换角色」进入商家端
2. **商品管理**：在「商品」页面添加、编辑、删除商品
3. **订单管理**：在「订单」页面查看和处理订单
4. **预订单管理**：在「预售接龙」页面创建和管理预订单
5. **数据统计**：在「数据中心」查看销售统计数据
6. **处理反馈**：在「系统通知」页面查看和回复用户反馈
7. **验证取货**：在「订单处理」页面输入取货码验证/直接使用扫一扫扫描买家取货二维码核销取货

### AI助手

- 在首页点击 AI助手 进入
- 可以询问商品信息、订单状态等问题
- AI会根据系统数据提供智能回复

## 技术实现

### 云函数实现

云函数采用 Node.js 编写，主要功能包括：

- 数据操作：对云数据库的增删改查
- 业务逻辑：处理订单、预订单、消息等业务逻辑
- 权限控制：验证用户身份和权限
- 触发器：响应数据库变更事件

### 前端实现

前端采用微信小程序原生开发，主要特点：

- 组件化开发：使用自定义组件提高代码复用性
- 页面分层：清晰的页面结构和导航
- 响应式设计：适配不同屏幕尺寸
- 数据绑定：使用小程序的数据绑定机制
- 云开发调用：直接调用云函数和云数据库

## 开发规范

### 代码规范

- 缩进：使用 2 个空格（在 project.config.json 中配置）
- 命名：使用驼峰命名法
- 注释：关键代码添加注释
- 文件结构：按功能模块组织文件

### 云函数规范

- 每个云函数独立目录
- 包含 config.json 配置文件
- 包含 package.json 依赖文件
- 主逻辑在 index.js 中实现

### 前端规范

- 页面文件：每个页面包含 js、json、wxml、wxss 文件
- 组件文件：每个组件包含 js、json、wxml、wxss 文件
- 样式规范：使用统一的样式命名和结构
- 逻辑分离：业务逻辑与 UI 逻辑分离
